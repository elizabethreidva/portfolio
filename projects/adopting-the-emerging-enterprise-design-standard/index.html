Adopting the Emerging Enterprise Design Standard

Problem

  • The emerging enterprise design system was the designated standard for all design work. However, the version in production was outdated and incomplete, while the updated version only existed in design specs, not in code. How can our business vertical achieve compliance with a design standard that exists in design but not in code with minimal development effort?

Objective

  • Propose a solution to balance compliance with the emerging state of the enterprise design system with utilizing existing coded components as much as possible.

Result

  • This project was still ongoing when I left the company. While we made some progress toward establishing a standard, the constantly shifting and poorly communicated design system release schedule made it challenging to move forward to tangible results.

Team

  • 1 Design Engineer (me)

Process

  • Review the design specs for the emerging version of the enterprise design system, and prioritize components that are needed for current and near future business vertical UX
  • For each component
    • Identify which existing coded libraries within the enterprise have an instance of the component. Consider both the legacy enterprise design system and any stable LOB-specific libraries.
    • Compare existing code instances against the latest enterprise design system design specs
    • Identify which existing code library is the closest match to the the enterprise design system
    • Document the differences between the closest match code component and the enterprise design
    • Determine the feasibility and level of effort to adapt the closest match code component to the enterprise design
  • Validate this approach with business vertical stakeholders and with the enterprise design system team

Challenges and Lessons Learned

  • The enterprise design system libraries were behind schedule, with no clear release roadmap. Designers used the latest specs, unaware that Tech partners lacked corresponding coded components. This disconnect between design and code was poorly communicated to stakeholders, which led to unrealistic expectations of what results were possible from this effort.
